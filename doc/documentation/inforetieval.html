Infomation and Data Retrevial Functions

These functions allow the end user to retrieve various pieces of infomation about an Image. While this infomation could be retrieved directly in some cases as the library is designed to deal with multiple mipmap levels and images these function represent a simple way to get that infomation without requiring a great deal of extra code.

getData

Function Prototypes

  • unsigned char * getData(Image const &img, int imgnumber = 0, int mipmaplvl = 0)
    Returns a pointer to the start of the image data for the requested image and mipmap level of a given Image object.

Return Value

  • A pointer to the start of the image data

Possible Known Exceptions

  • none

Other Details / Notes

  • This function doesnt to any checking as to the validity of the image number nor the mipmap level, this is left to the user

getWidth

Function Prototypes

  • int getWidth(Image const &img, int mipmaplvl = 0)
    Returns the width of the image at the given mipmap level

Return Value

  • Width of the imageat the requested mipmap level

Possible Known Exceptions

  • None

Other Details / Notes

  • The image number isnt required as the library assumes all images have the same dimensions (true of DDS images)

getDepth

Function Prototypes

  • int getDepth(Image const &img, int mipmaplvl = 0)
    Retieves the depth of the image at the given mipmap level

Return Value

  • depth of the image at the requested mipmap level

Possible Known Exceptions

  • None

Other Details / Notes

  • The image number isnt required as the library assumes all images have the same dimensions (true of DDS images)
  • 2D images will return a depth of 1

getSize

Function Prototypes

  • int getSize(Image const &img, int mipmaplvl = 0)
    Retieves the size in bytes of the image at the given mipmap level

Return Value

  • The size in bytes of the image at the given mipmap level

Possible Known Exceptions

  • None

Other Details / Notes

  • The image number isnt required as the library assumes all images have the same dimensions (true of DDS images)
  • The function correctly works out the size for DXTn compressed data
